# 工具箱小程序 · 产品规划（基准版本：v1.2.0）

> **文档索引**：[../README.md](../README.md)  
> **文档性质**：在 **v1.2.0** 规划交付或并行迭代背景下，新增 **「休闲游戏」模块** 的里程碑方案；首发游戏为 **贪吃蛇**。  
> **前置文档**：[产品规划-v1.2.0.md](./产品规划-v1.2.0.md) · [产品规划-v1.1.0.md](./产品规划-v1.1.0.md)  
> **命名说明**：本文件标识 **`v1.3.0`**（**轻度娱乐增量**：游戏分包 + 首款单机小游戏）。再往后可另存 **`产品规划-v1.4.0.md`** 等。

## 一、文档摘要

| 项 | 内容 |
|----|------|
| **前置能力** | 工具注册表、分包、`applyThemeForToolPage`、本地存储等既有基建 |
| **本里程碑主线** | **v1.3.0**：上线 **游戏模块壳 + 贪吃蛇（单机）** |
| **产品定位补充** | 在「效率工具」心智之上增加 **短时放松** 场景，提升打开频次与停留（非主业，需克制复杂度） |
| **本版原则** | **单机、零账号、无付费**；包体与审核可控；**不与工具主业抢叙事**（首页仍为工具优先） |

---

## 二、背景与机会

| 点 | 说明 |
|----|------|
| **动机** | 工具使用多为任务导向；轻量小游戏可提高 **趣味性与回访**，便于亲友传播时的记忆点。 |
| **风险意识** | 微信对类目与内容有监管；游戏过重易被视作「游戏类小程序」，需在 **类目与服务范围** 上与主体资质对齐（研发上架前必审）。 |
| **取舍** | 首期只做 **规则极简、体量小** 的贪吃蛇，验证分包与体验流程；不做排行榜云同步、不做对战。 |

---

## 三、版本目标（一句话）

**在不稀释「工具箱」主心智的前提下，用独立分包提供一个「小游戏」入口，首发贪吃蛇单机版，本地记分、完整一局闭环。**

---

## 四、信息架构与入口（建议）

### 4.1 分类与注册表

| 建议 | 说明 |
|------|------|
| **新分类** | 新增 `games`（或 `leisure`），名称示例：**「休闲」** / **「小游戏」**，图标示例：**🎮**。 |
| **工具条目** | `id: snake`，名称：**贪吃蛇**，描述：**经典单机，触控转向**，路径指向分包页（见工程 §）。 |
| **首页** | 保留现有工具宫格逻辑；可在「猜你喜欢」下方或单独区块增加 **「小游戏」横滑或一行入口**（可选，避免首屏过长则以分类 Tab 为主）。 |

### 4.2 心智边界（文案层）

- Tab / 分类名称强调 **「休闲」「单机」**，避免「网游」「竞技」等敏感联想（除非 future 真做）。  
- 游戏页 Footer 可加一句：**适度游戏，注意休息**（轻量提示即可）。

## 五、贪吃蛇 MVP 需求（产品）

### 5.1 玩法（必选）

| ID | 需求 | 验收要点 |
|----|------|----------|
| G1 | **网格平面贪吃蛇**，吃食物变长，撞墙或撞自身则游戏结束 | 规则符合大众直觉；无明显漏洞（穿墙等除非刻意做彩蛋） |
| G2 | **触控转向**：滑动手势改变前进方向（上下左右） | 误触率可接受；可选 **虚拟方向键** 作为无障碍/B方案 |
| G3 | **暂停 / 继续** | 来电或切换会话回来时状态策略：暂停或本局结束（产品择一写死） |
| G4 | **重新开始** | 任意时刻可新开一局 |
| G5 | **本局得分展示** | 与长度或吃掉个数一致，规则对用户可读 |

### 5.2 进度与激励（必选）

| ID | 需求 | 验收要点 |
|----|------|----------|
| G6 | **本地最高分** | `wx.setStorageSync`，键名建议 `tool_snake_highscore_v1`；进入页可见历史最佳 |
| G7 | **游戏结束页** | 展示本局得分、是否破纪录、按钮「再玩一局」「返回」 |

### 5.3 体验（建议）

| ID | 需求 | 优先级 |
|----|------|--------|
| G8 | **难度或速度档位**（如：慢 / 标准 / 快） | P1 |
| G9 | **与全局明暗主题一致**（沿用 `applyThemeForToolPage` 或等价变量） | P0 |
| G10 | **简短玩法说明**（首次进入或「？」入口） | P1 |

### 5.4 明确不做（本期）

- 联机、排行榜（云端）、道具付费、广告 SDK（除非后续单独立项）。  
- 复杂皮肤商店（可预留常量占位，不阻塞 MVP）。

## 六、工程与分包建议（供研发对齐）

| 项 | 建议 |
|------|------|
| **分包** | 新建 **`package-game`**（或 `package-leisure`），游戏统一放此分包，控制主包体积。 |
| **实现路径** | Canvas 2D / 小游戏框架二选一由研发评估；首期贪吃蛇 **帧率与耗电** 需在真机验证。 |
| **路由示例** | `/package-game/snake/index`（仅示意）。 |

---

## 七、合规与审核自检（产品牵头）

| 项 | 说明 |
|------|------|
| **服务类目** | 核对现行微信类目：**休闲娱乐 / 工具** 等与主体经营范围一致；若仅有「工具」类目，需确认贪吃蛇是否可被接受（以审核口径为准）。 |
| **用户协议与隐私** | 单机本地记分一般不采集个人信息；若未来接广告或排行榜再更新条款。 |
| **适龄提示** | 轻度益智通常无强制版号要求，仍以平台当下规则为准；文档交付时需 **法务/运营复核**。 |

---

## 八、成功指标（轻量）

| 指标 | 含义 | 备注 |
|------|------|------|
| **游戏页 UV / PV** | 是否有人玩 | 可与现有匿名统计策略一致（若有） |
| **平均单次停留时长** | 是否构成「玩一局」 | 抽样观察即可 |
| **崩溃与卡顿投诉** | Canvas 性能 | 首发一周内重点盯 |
