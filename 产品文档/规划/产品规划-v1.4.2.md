## 一、目标

在小程序内提供 **大模型文字对话**，形态上接近「学习助手」：**无语音、无录音**；用于释疑、造句、近义词辨析等，**降低**相对语音链路的 **审核与工期** 压力，并验证 **token 成本与限流**。

---

## 二、与单词本的关系

| 策略 | 说明 |
|------|------|
| **上下文携带** | 从详情页可 **「带着当前词进对话」**（预填或系统消息注入 headword + 释义摘要）。 |
| **独立入口** | 「学单词」模块内 **「对话」Tab** 或二级页；默认系统提示词偏向 **英语学习助手**（可配置）。 |
| **自由模式** | 可选「自由提问」与「围绕当前词」切换；**自由模式**需在 UI 标明 **AI 生成**。 |

---

## 三、功能范围（MVP）

| ID | 需求 | 验收 |
|----|------|------|
| T1 | 多轮文字对话，展示 **用户 / AI** 气泡 | 滚动到底、长文换行正常 |
| T2 | **清空会话** / **新会话** | 不污染上一轮上下文（或明确「新会话」清上下文） |
| T3 | **流式输出（可选 P1）** | 首字延迟可接受；失败 Toast + 重试 |
| T4 | **敏感与违规拦截** | 服务端或网关策略；拒绝时友好文案 |
| T5 | **登录后可用（推荐）** | 便于限流与审计；若允许游客需单独风险评估 |

**不做（本期）**：语音输入输出、对话分享为「可编辑转发」的误导形态（若分享需带 AI 声明）。

---

## 四、合规与披露

| 项 | 说明 |
|------|------|
| **生成式内容标识** | 对话区域持久或首次显著提示 **「内容由 AI 生成」**。 |
| **隐私政策** | 说明 **对话是否存储、存多久、是否用于模型训练**（建议默认 **不用于训练**）。 |
| **类目** | 与主体资质、微信类目 **运营 + 法务** 核对。 |

---

## 五、成本与稳定性

| 项 | 说明 |
|------|------|
| **限流** | 每用户每日条数 / 每分钟条数；超额提示。 |
| **降级** | 模型超时 → 提示稍后重试；可选 **备用模型**（研发定）。 |

|------|------|

| 单位时间 token 成本 | 财务可承受 |
