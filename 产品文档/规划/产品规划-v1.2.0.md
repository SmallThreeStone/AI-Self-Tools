## 二、v1.2.0 版本目标（一句话）

**让用户「用得安心」：偏好与关键本地数据可备份与迁移；外观与系统一致；分享出去时能一眼看出「来自哪个工具」。**

## 三、需求说明（建议优先级）

### 3.1 P0 —— 配置导出 / 导入（备份与换机）

| ID | 需求 | 用户故事 | 验收要点 |
|----|------|----------|----------|
| D01 | **导出扩展** | 我在「我的」里一键导出，得到的不仅是收藏列表，还能带上可选工具数据 | 导出物为 **单一 JSON**（或分包压缩策略若体积过大再评审）；内含 **`schemaVersion`** 字段便于后续演进 |
| D02 | **导入与合并策略** | 换手机或误删后我能恢复 | 支持从聊天记录选文件导入；**合并策略**需明确：默认「合并收藏去重」还是「覆盖」——产品上建议默认合并 + 二次确认覆盖 |
| D03 | **数据范围清单（面对面公示）** | 我知道备份里包含什么 | 「我的」页展示可导出字段说明：收藏 id、最近使用、主题偏好、以及已接入工具的白名单存储键（逐项列出，避免用户以为含相册原图） |
| D04 | **异常处理** | 文件损坏或版本不兼容时不崩溃 | 校验失败 Toast + 不写脏数据；旧版 schema 可做 **只读迁移** 或提示「部分字段已废弃」 |

**建议最小接入集合（研发核定）**：`toolbox_favorites`、`toolbox_recent`、`toolbox_theme`、`toolbox_usage`（若有）；以及各工具已约定的 `tool_*_v1` 键——需维护一张 **允许导出的键注册表**，防止把临时缓存打进去。

**不做**：云端自动同步、跨设备实时推送。

---

### 3.2 P1 —— 深色模式：跟随系统与手动并存

| ID | 需求 | 说明 | 验收要点 |
|----|------|------|----------|
| T01 | **三态策略** | 用户可选：**跟随系统** / **浅色** / **深色** | 与现有「我的」里主题切换 UI 融合；持久化到本地 |
| T02 | **监听系统主题** | 在「跟随系统」下，系统深浅变化时小程序同步 | 使用 `wx.onThemeChange`（或当前基础库推荐 API）；进入前台时校正一次 |
| T03 | **导航栏与 TabBar** | 切换时不闪屏、不残留错误的前景色 | 与现有 `applyNavigationBarTheme` / 全局样式变量对齐；必要时 TabBar 配置限制内需注明取舍 |

**边界**：若基础库或机型差异导致跟随异常，降级为「仅手动」并在说明文案中注明。

---

### 3.3 P1 —— 分享卡片差异化（拉新与辨认度）

| ID | 需求 | 说明 | 验收要点 |
|----|------|------|----------|
| S01 | **工具页分享** | 从某一工具页转发，标题含 **工具名称** 或固定句式「工具箱 · xxx」 | `onShareAppMessage` 在分包工具页返回定制 `title`；`path` 可带 `toolId` 查询参数（首页 **onLoad 解析并跳转**，需防 XSS/非法 id） |
| S02 | **首页 / 分类默认分享** | 保持通用标题与简介 | 不与工具页冲突 |
| S03 | **可选**：分享图 `imageUrl` | 若已有品牌图或工具占位图再启用；无图则用默认 | 避免大图拖慢分享面板 |

**合规**：分享文案避免出现诱导点击、虚假抽奖等审核敏感表述。

---

### 3.4 P2 —— 增值项（资源允许时插入）

| ID | 需求 | 说明 |
|----|------|------|
| V01 | **导入导出进度** | 大数据键时显示 loading，防止误以为卡死 |
| V02 | **极简迁移向导** | 首次检测到「空收藏 + 无最近」时可提示「是否从备份导入」（仅一次或可跳过） |

## 四、与后续版本（v1.3.0）的边界

以下内容 **默认不放入 v1.2.0**，避免范围失控：

- **UGC / 自定义题库上传**、社区化；
- **订阅消息**（除非与备份提醒单独立项）；
- **小程序插件化**、第三方统计 SDK 深度集成。

后续单独输出 **`产品规划-v1.3.0.md`**（与本文件同目录）时再展开。

## 五、成功指标（v1.2.0）

| 指标 | 含义 | 期望 |
|------|------|------|
| **备份功能触达** | 「导出」按钮点击完成率（若有埋点） | 小众功能，不设过高 KPI，以 **零事故** 为先 |
| **导入失败率** | 解析失败 / 用户投诉「恢复不了」 | 接近 0；错误文案可读 |

| **分享带来的回访（可选）** | 后台场景值 / 分享链路透传（微信自带） | 环比持平或略升即可 |

## 六、风险与依赖

| 风险 | 应对 |
|------|------|
| **导入恶意 JSON** | 严格 schema 校验；限制深度与数组长度；禁止 `eval` |
| **导出含敏感本地数据** | 仅白名单键；密码类工具若导出需二次确认 |
| **跟随系统主题与自定义 TabBar** | 提前查基础库与官方限制；准备降级方案 |
| **分享 path 参数被篡改** | `toolId` 必须在注册表中存在才跳转 |

---

## 七、开放问题（评审闭合）

1. **导出文件是否加密或压缩**：明文 JSON 更易调试；若加密则涉及密钥存放（慎用）。  
2. **合并 vs 覆盖的默认策略**：影响老用户恢复体验，需一次性定稿。  
3. **分享落地**：是否必须首页中转，还是允许直达分包页（影响路径与首次加载）。
