---

## 一、目标

提供 **类豆包** 的 **语音对话**：**语音输入 →（ASR）→ 大模型 →（TTS）→ 语音播报**，界面可 **同步展示文字 transcript**；与 **单词本** 弱联动（默认学习向系统提示词 + 可带当前词上下文）。

---

## 二、用户能感知到的能力

| 能力 | 说明 |
|------|------|
| **语音输入** | **按住说话** 或 **点击开始 / 停止录音**（二选一或并存）；超时自动结束；支持 **打断正在播放的 TTS**（P1）。 |
| **智能回复** | 同 v1.4.2 所用模型或升级版本；系统提示词偏 **口语陪练 / 纠音 / 用词建议**。 |
| **语音输出** | AI 回复 **TTS 播报**；语速调节 **P1**。 |
| **对话记录** | 默认 **本会话内可见**；是否落库、保留时长 **必须写入隐私政策**并由产品定稿。 |

---

## 三、与单词本 / 文字对话的衔接

| 策略 | 说明 |
|------|------|
| **入口** | 「学单词」内 **「语音对话」** 与「文字对话」并列或二级进入，避免与工具首页混淆。 |
| **带词进入** | 同 v1.4.2：从词条详情进入时可注入当前词。 |
| **自由闲聊** | 单独模式开关 + **AI 生成** 披露；审核友好文案。 |

---

## 四、权限与合规（必选）

| 项 | 说明 |
|------|------|
| **录音** | `scope.record`；首次前 **用途说明**；录音文件 **上传最小化**（如仅上传识别所需格式与时长）。 |
| **深度合成 / 生成式服务** | 遵守平台与监管要求；**法务评审** 后再排期。 |
| **登录** | **强烈建议仅登录用户可用**（限流、审计、未成年策略若需要）。 |

---

## 五、技术链路（产品理解用）

**小程序录音 → 上传 / 流式 ASR → LLM → TTS → 小程序播放**；会话 ID、鉴权、限流、超时均在 **服务端**。

**弱网**：ASR / TTS 失败重试；可选 **仅返回文字** 降级。

---

## 六、分期（本册内可选）

| 阶段 | 内容 |
|------|------|
| **V1** | 按住说 + 非流式 ASR + LLM + TTS 串行，延迟可接受即可 |
| **V2** | 流式 TTS / 流式 ASR、打断播报、音色可选 |

---

## 七、成功指标

| 指标 | 说明 |
|------|------|
| 语音会话人均轮次、完成率 | 是否形成「说—听」闭环 |
| ASR / TTS 失败率 | 体验与厂商 SLA |
| 审核与客诉（录音、AI） | 相对 v1.4.2 增量可控 |

---

## 八、风险

| 风险 | 应对 |
|------|------|
| 成本陡增 | 严格限次、仅登录、高峰排队提示 |
| 隐私敏感 | 默认不用于训练、明示保留期限 |
| 延迟差 | 预期管理 + 文字兜底 |

---

## 九、开放问题

1. 厂商选型：**火山豆包** 是否与主体合同一致？  
2. 是否允许 **仅语音不展示文字**（不推荐，审核与无障碍差）？  
3. 会话 **最长录音时长** 上限（如 60s）？
