## 〇、背景与目标


| 项      | 说明                                                                    |
| ------ | --------------------------------------------------------------------- |
| **现状** | 单词本以列表 + 详情为主；例句展示多为纯文本，**目标词在句中不突出**；缺少 **重点词沉淀** 与 **扩展例句** 的个性化能力。 |
| **诉求** | ① **我的收藏**：对重点学习单词一键收藏、集中复习；② **例句内目标词高亮**；③ **自定义添加例句**，扩大语境学习面。     |
| **原则** | 默认 **本地存储**；与内置例句、用户整词（v1.4.10）字段模型 **可合并演进**，避免三套互不关联的数据结构。          |

## 一、「我的收藏」功能

### 1.1 产品定义

| 点 | 说明 |
|----|------|
| **收藏对象** | **词条**（以 `wordId` 或 `headword+category` 唯一标识）；内置词与用户词 **均可收藏**（若用户词尚未上线，可先仅内置）。 |
| **状态** | 布尔：已收藏 / 未收藏；在 **列表卡片** 与 **详情页** 均提供 **星标 / 心形** 切换。 |
| **入口** | **单词本内** 第四类筛选：**「全部 \| 日常 \| 外贸 \| 仅收藏」**（已定稿，见 §八）。 |
| **空态** | 无收藏时展示插图文案 + 引导「在词条详情点亮星标」。 |


### 1.2 数据与排序


| 项        | 建议                                                                    |
| -------- | --------------------------------------------------------------------- |
| **存储**   | `learn_favorites_v1`: `string[]` 存词条 id；或 `{ id, ts }[]` 支持「按收藏时间」排序。 |
| **排序**   | 默认 **最近收藏在上**；可选按字母序（P1）。                                             |
| **取消收藏** | 与添加同一入口；列表与详情同步，无需杀进程。                                                |


### 1.3 与「日常 / 外贸」的关系


| 策略     | 说明                                                  |
| ------ | --------------------------------------------------- |
| **推荐** | 「我的收藏」为 **跨分类合集**；列表每条 **角标** 标明原属「日常」或「外贸」，避免语境丢失。 |
| **可选** | 收藏筛选下再分子 Tab 日常/外贸（P1，首版可不做）。                       |

## 二、例句中「目标词」高亮

### 2.1 展示规则


| 规则       | 说明                                                         |
| -------- | ---------------------------------------------------------- |
| **高亮对象** | 当前详情页所属词条的 **headword**（英文）；在 **每条例句英文文本** 中匹配并高亮。         |
| **视觉**   | 建议使用 **品牌色底 + 加粗** 或与正文对比度足够的背景条；**明暗主题** 下均可读（见设计 token）。 |
| **大小写**  | 匹配时 **不区分大小写**；展示保留原文大小写。                                  |


### 2.2 边界（产品需接受或二期解决）


| 场景                          | 处理                                                              |
| --------------------------- | --------------------------------------------------------------- |
| **词形变化**（如 study / studied） | MVP 可仅 **精确子串匹配** 整词；**词干高亮** 为 P2（需形态分析或手动标注 `highlightSpan`）。 |
| **一词多义多例句**                 | 每条例句独立高亮；同句出现两次该词则 **两处均高亮**。                                   |
| **例句为中文或中英混排**              | 若句中无英文 headword，则 **不高亮**（不强行匹配中文释义）。                           |


### 2.3 与自定义例句的关系

用户自定义例句（§三）与内置例句 **使用同一套渲染组件**：均传入 `sentence` + `headword` → 输出带高亮的富文本（或 `rich-text` 节点数组）。

## 三、自定义添加例句

### 3.1 产品定义


| 点           | 说明                                                            |
| ----------- | ------------------------------------------------------------- |
| **作用域**     | 从 **某词条详情页** 进入「添加例句」；例句 **挂载在当前词条** 下，与内置例句 **同列表展示**（区分来源）。 |
| **字段（MVP）** | **英文例句**（必填）；**中文参考译（选填）**；可选 **备注**（如「来自某客户邮件」）。             |
| **条数** | **不设硬性条数上限**（已定稿，见 §八）；通过 **置顶** 突出重点句；可选 **软提示**（如单词条超过 50 条时提醒「过多可能影响列表性能」）由研发评估是否要做。 |
| **置顶** | 用户例句支持 **置顶**（已定稿）：置顶项在「我的例句」分组内 **固定最前**；多置顶时按置顶时间倒序。 |


### 3.2 展示与操作


| 能力        | 说明                                                          |
| --------- | ----------------------------------------------------------- |
| **列表**    | 详情页例句区：**内置** 与 **我的例句** 分组标题，或统一列表 + **「我的」小角标**。          |
| **高亮**    | 用户例句同样执行 **§二** 高亮逻辑。                                       |
| **播放** | **优先为用户例句提供整句 TTS**（与内置一致）；若首期接口/成本阻塞，可临时 **仅内置句可播** 并在例句旁灰字说明，**P1 必须补齐**（已定稿方向，见 §八）。 |
| **改 / 删** | 仅 **用户添加** 的例句可编辑、删除；内置只读；二次确认删除。                           |


### 3.3 数据模型（建议）

在词条或用户扩展结构中增加 `userExamples[]`：


| 字段          | 说明       |
| ----------- | -------- |
| `id`        | 本地唯一 id  |
| `textEn`    | 英文例句全文   |
| `textZh`    | 可选中文     |
| `note`      | 可选备注     |
| `createdAt` | 时间戳，便于排序 |
| `pinned` | 可选：`boolean` 或 `pinnedAt` 时间戳，支持 **置顶**（见 §八） |


与 **v1.4.10 用户整词** 的关系：若用户词整条是用户自建，例句可挂在该条 `userWord` 下；若为内置词，则 `userExamples` 挂在 **全局扩展表 keyed by `wordId`（与内置 id 一致）**——**研发选型**，产品要求：**从详情进入的例句必须能关联到当前词**。

**收藏与例句 id 体系（产品侧建议，已定稿方向）**：见 **§八 · 决议 5**。

## 四、信息架构（示意）

```
单词本
├── [全部 | 日常 | 外贸 | 仅收藏]  ← 筛选或 Tab，与现有搜索、列表兼容
├── 搜索栏（行为与 §八 一致：在当前 **筛选范围** 内检索，如「仅收藏」下只搜已收藏词；「全部」下搜全量）
└── 词条详情
    ├── 星标（收藏）
    ├── 释义区
    └── 例句区
        ├── 内置例句（只读，高亮 headword）
        ├── 我的例句（可增删改，高亮 headword）
        └── [+ 添加例句]
```

## 五、分期建议


| 阶段      | 内容                                          |
| ------- | ------------------------------------------- |
| **MVP** | 收藏 / 取消 + **「仅收藏」筛选** + 例句高亮（精确匹配）+ 自定义例句增删改（英文必填）+ **用户例句置顶** |
| **P1**  | 用户例句 **整句 TTS**（若 MVP 因成本/接口未上，本项为必补）；收藏按字母序；导出包含收藏与自定义例句；单词条例句 **软提示**（条数过多） |
| **P2**  | 词形变化智能高亮；批量导入例句                             |

## 六、验收标准（产品侧）

1. 任意词条可 **收藏/取消**，在「仅收藏」视图下 **仅出现已收藏词**。
2. 内置与用户例句中，**headword 子串** 均以约定样式 **高亮**（含用户新加例句）。
3. 为某词添加 2 条自定义例句后，详情 **立即展示**；删除一条后另一条仍在；内置例句 **不可删**。
4. 收藏状态在 **列表与详情** 一致切换，无脏数据。
5. **不设硬性条数上限** 时，超大量例句下列表仍 **可滚动、不白屏**；若实现软提示，需在超阈值时出现且可关闭。  
6. 用户例句 **置顶** 后，刷新详情仍保持 **置顶顺序**。

## 八、产品决议（已根据你方答复整理）

以下取代原「待确认清单」，作为 **开发与验收依据**。

| # | 议题 | 决议 |
|---|------|------|
| 1 | **「我的收藏」入口** | **采用** 单词本内 **第四筛选**：「全部 \| 日常 \| 外贸 \| 仅收藏」。 |
| 2 | **例句高亮策略** | **MVP 采用** 对 `headword` 的 **精确子串匹配**（不区分大小写）；词形变化智能高亮保留 **P2**。 |
| 3 | **用户例句整句 TTS** | **能做则做**：与内置例句一致支持朗读；若首期存在接口/成本/工期阻塞，允许 **临时仅内置句可播**（需在 UI 弱提示），**P1 必须** 为用户例句补齐 TTS。 |
| 4 | **用户例句条数上限** | **不设硬性上限**；接受存储与列表变长的风险。工程侧可做 **性能保护**（虚拟列表、懒渲染）及可选 **软提示**（如单词条 >50 条时提示「过多可能影响体验」），**不阻止保存**。 |
| 5 | **用户例句置顶** | **纳入 MVP**：用户例句支持 **置顶 / 取消置顶**；置顶项在「我的例句」区域内 **始终排在最前**（多置顶按置顶时间倒序）。 |
| 6 | **收藏 id 与用户词 id 体系** | **产品建议（请研发输出《字段表》评审签字）**：<br>• **内置词**：使用稳定 **`wordId`**（与词表 JSON 主键一致）。<br>• **用户自建词**（v1.4.10）：使用 **`userWordId`**（UUID 等）。<br>• **`learn_favorites_v1`**：存 **`targetId` + `targetType`**（`builtin` \| `user`），或存统一 **`canonicalWordId`**（由研发定义拼接规则），避免仅用 `headword` 导致跨分类歧义。<br>• **自定义例句**：以 **`targetId` + `targetType`** 外键挂载至当前词条，与收藏列表解析一致。 |
