| **缺口** | 词条详情中 **例句**（含 **内置例句** 与 **用户自定义例句**）需支持 **整句英文语音播放（TTS）**，与「单词朗读」形成完整听读闭环。                                                     |
| **目标** | **同一套交互与同一套播放管线**：每条可朗读的例句均有 **播放 / 重播 / 播放中态**；内置与用户句 **无区别对待**（仅文本来源不同）。                                                       |

## 一、范围（必须做）


| ID  | 场景          | 验收要点                                                        |
| --- | ----------- | ----------------------------------------------------------- |
| S1  | **内置例句**    | 每条内置英文例句旁有 **播放** 控件；点击后 **朗读整句**（英文）。                      |
| S2  | **用户自定义例句** | 「我的例句」中每条 **用户添加** 的英文例句同样 **整句 TTS**；置顶、排序不改变播放能力。         |
| S3  | **与单词播放共存** | 正在播放 **单词** 时点击 **例句**，应 **停止前者再播例句**（或统一为「全局单一音频实例」，避免叠音）。 |
| S4  | **弱网 / 失败** | TTS 请求失败时 **Toast + 可重试**；不出现无限 loading。                    |
| S5  | **每日次数上限** | **首期启用** 例句（或与单词 TTS **共用配额**，由研发定）**每日播放次数上限**；触顶 **Toast** 说明原因与恢复时间（如次日）；阈值与开关 **服务端可配置**，**预留后期放宽或分档开放**（见 §八 决议）。 |


**不在本期（可另开优化）**：中文例句整段朗读、多发音人自选、例句跟读打分、后台续播。

## 二、交互与 UI（建议）


| 点        | 说明                                                        |
| -------- | --------------------------------------------------------- |
| **控件位置** | 每条例句右侧或句尾 **喇叭图标**；与现有「单词朗读」风格统一（尺寸、颜色 token）。            |
| **状态**   | 未播 / 加载中（转圈或文案「合成中…」）/ 播放中（可变为「停止」或再次点击停止）/ 失败（图标灰 + 重试）。 |
| **连续点击** | 同一句 **连点** = **从头重播**；不同句点击 = 切曲并播新句。                     |
| **长句**   | 句长超过阈值（如 300 字符）时仍允许播放，但可 **弱提示**「较长，加载可能更久」（可选）。         |

## 三、技术产品要求（供研发对齐，不定具体厂商）


| 项        | 说明                                                                                                                     |
| -------- | ---------------------------------------------------------------------------------------------------------------------- |
| **语种 / 音色** | 例句 TTS 默认 **英文**（美/英音与全局「学单词」配置一致即可）；与 **单词朗读音色不强制一致**（可不同音色/不同引擎），以降低耦合与实现成本；若同链路零成本复用则允许一致。 |
| **实现路径** | **实时 TTS API**（文本上云合成返回音频 URL / base64）或 **预置音频 URL**（仅内置句适用）；**用户句一般为实时合成**。                                          |
| **缓存**   | 对 **同一 `sentenceId` + 文本 hash`** 的合成结果 **本地缓存 URL 或文件**（`wx.getFileSystemManager` / 临时文件），减少重复扣费与等待；用户编辑例句后 **使缓存失效**。 |
| **鉴权 / 频控** | 若走自建后端：需 **限流**、防刷；错误码映射为用户可读文案。**每日播放上限**（§一 S5）在网关或业务层计数，**用户维度**（`openid` 等）统计，与风控策略对齐。 |
| **隐私**   | 仅将 **待朗读的英文句子** 发往 TTS 服务；不在日志中落全量用户笔记（合规最小化）。                                                                         |

## 五、验收标准（产品侧）

1. 任意 **内置** 英文例句点击播放，**10s 内**（常网）可听到声音或明确失败提示。
2. 任意 **用户自定义** 英文例句点击播放，行为与内置 **一致**（含停止、重播）。
3. **离开详情页**（返回、切 tab、进子页等导致词条详情卸载）：例句播放 **一律停止**（已定稿，不做后台续播）。
4. 用户 **修改** 例句文案后，再点播放，内容为 **新文案**（缓存失效生效）。
5. 无英文内容（纯中文例句）的条目：**不展示** 播放或点击后 Toast「暂不支持朗读该句」。
6. 达到 **当日 TTS 次数上限** 时：再次点击播放有 **明确 Toast**（含「今日次数已用完」类语义）；次日或配置重置后恢复；后台 **可调阈值 / 可关闭上限**（便于后期放宽）。

## 六、分期（可选）


| 阶段      | 内容                               |
| ------- | -------------------------------- |
| **MVP** | 内置 + 用户例句实时 TTS、单实例互斥、失败重试、**离开详情页一律停止**、**每日播放上限（可配置）** |
| **P1**  | 本地缓存；若产品后续要求「例句音色与单词一致」再做设置项联动（当前 **不强制**） |
| **P2**  | 预生成内置句离线包、降低流量；上限 **分会员 / 活动** 放宽 |

## 七、非目标

- 例句 **中文** TTS、**自动翻译** 后朗读。  
- 用户 **上传录音** 替代 TTS。  
- **后台播放** 与锁屏控制条（小程序能力受限，默认不做）。

## 八、产品决议（原「待确认」已合并）

以下为用户答复后的 **最终产品口径**，与 §一～§六 已交叉修订一致。

| 议题 | 决议 | 落地 |
| ---- | ---- | ---- |
| 离开详情页时是否停止例句播放 | **是**，一律停止 | §一 场景隐含；§五-3；§六 MVP |
| 例句英文音色是否与单词朗读一致 | **不强制一致** | §三「语种 / 音色」；§六 P1 说明 |
| 是否每日播放次数上限 | **首期要做**；**后期可通过配置或运营策略放宽 / 分档** | §一 S5；§三「鉴权 / 频控」；§五-6；§六 MVP / P2 |

### 8.1 优化项总表（便于排期 / 拆任务）

| 编号 | 优化项 | 说明摘要 | 优先级 |
| ---- | ------ | -------- | ------ |
| O-01 | 内置例句整句 TTS | 喇叭入口、加载 / 播放 / 失败态 | P0 |
| O-02 | 用户自定义例句整句 TTS | 与内置同一套交互与接口策略 | P0 |
| O-03 | 全局单一音频实例 | 单词播放与例句播放 **互斥**，后点覆盖先播 | P0 |
| O-04 | 离开详情页停止播放 | 详情卸载即 `stop`，不后台续播 | P0 |
| O-05 | 弱网与失败体验 | Toast、可重试、无死循环 loading | P0 |
| O-06 | 文案变更后缓存失效 | 编辑例句后再播为新文本 | P0 |
| O-07 | 非英文例句无播放或弱提示 | 纯中文句：隐藏播放或 Toast | P0 |
| O-08 | 每日 TTS 次数上限 | 可配置阈值；触顶 Toast；预留后期放宽 | P0 |
| O-09 | 例句 TTS 音色 | **独立于**单词朗读音色亦可 | P0（口径） |
| O-10 | 本地合成结果缓存 | 降延迟与费用（§六 P1） | P1 |
| O-11 | 内置句离线 / 预生成 | 省流量（§六 P2） | P2 |
