## 〇、背景与问题

| 项 | 说明 |
|----|------|
| **现状** | 单词本（日常 / 外贸 Tab + 搜索 + 列表卡片）当前为 **内置写死词表**，用户无法维护自有词条。 |
| **诉求** | 支持用户对词条 **自定义增、删、改**，避免数据完全依赖发版更新。 |
| **边界** | 本期聚焦 **本地个人词库**；**不与** v1.4.2 / v1.4.3 的云端对话强绑定（可预留字段）。 |

---

## 一、产品目标（确认后开发）

1. **增**：用户可在「日常」「外贸」分类下 **新增词条**（至少：英文词头 + 中文释义；若详情页已有例句/读音能力，新增时可 **选填** 或后续在详情里补全）。  
2. **删**：用户可删除 **自己添加** 的词条；内置词是否允许删除见 **§四 策略选型**（需你拍板）。  
3. **改**：用户可编辑 **自己添加** 的词条内容；内置词是否允许改见 **§四**。  
4. **搜**：现有「中/英文搜索」在合并展示时，**用户词与内置词** 一并参与检索，规则一致。

---

## 二、信息架构与入口（建议）

| 方案 | 说明 | 推荐度 |
|------|------|--------|
| **A. 列表顶栏「+ 添加」** | 与搜索同排或右侧图标按钮，进入「添加词条」表单页 | 高，符合移动端习惯 |
| **B. 长按卡片** | 长按弹出 **编辑 / 删除**（或底部 action sheet） | 中高，需发现成本提示 |
| **C. 详情页内编辑** | 从列表进详情后，对用户词显示「编辑」入口 | 高，与「补全例句」同路径 |

**建议组合**：**A + C**（添加用顶栏；改删在详情页最清晰）；**B** 作为补充（列表页快捷删）。

**视觉区分（强烈建议）**：用户自建词条卡片角标 **「我的」** 或小图标，与内置词区分，避免用户误以为删了「官方词」。

## 三、数据与合并策略（需评审拍板）

### 3.1 推荐方案：**内置词库 + 用户词库分层合并**

| 数据源 | 存储 | 展示 |
|--------|------|------|
| **内置词** | 分包静态资源 / 只读 JSON | 列表与搜索可读；**默认不可删**（可选「隐藏」见下） |
| **用户词** | `wx.setStorage`（或本地 SQLite 若已引入） | 与内置词 **同列表合并排序**；**可增删改** |

**合并规则**：同一 `headword`（忽略大小写）同时存在内置与用户词时：

| 策略 | 说明 |
|------|------|
| **优先生效用户词（推荐）** | 列表只展示一条，以用户修改为准，详情页标明「已覆盖内置释义」 |
| **并列展示** | 两条都出现，易混乱，不推荐 |

### 3.2 用户词条最小字段（MVP）

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 系统 | 本地 UUID 或时间戳+随机串 |
| `headword` | 是 | 英文词头；trim；长度上限（如 48） |
| `zh` | 是 | 主中文释义 |
| `category` | 是 | `daily` \| `trade`，与当前 Tab 默认一致，可表单内切换 |
| `source` | 系统 | `user` \| `builtin`，用于控制删改权限与角标 |
| `examples` / `phonetic` / 音频 | 否 | 与 v1.4.1 详情结构对齐，可二期在编辑页补全 |

### 3.3 存储键（示例，研发可调整）

| 键 | 含义 |
|----|------|
| `learn_user_words_v1` | 用户词条数组 JSON |

**备份**：若已有「配置导出」能力，需将用户词纳入 **可导出白名单**（另文/另迭代）。

## 四、内置词是否允许删改（开放问题 → 请你确认）

| 选项 | 体验 | 风险 |
|------|------|------|
| **A. 内置只读（推荐默认）** | 用户只能改自己的词；内置永远存在 | 无法去掉「不需要的官方词」 |
| **B. 内置可「隐藏」** | 写入 `hidden_builtin_ids` 本地集合，列表与搜索过滤 | 实现与搜索略复杂；恢复入口需有 |
| **C. 内置可改** | 与内置包更新冲突，**不推荐** | 升级发版覆盖逻辑难处理 |

**建议默认**：**A**；若你强需求「不要看到某内置词」，采用 **B**。

## 五、交互与校验

| 场景 | 规则 |
|------|------|
| **新增** | 空词头/空释义禁止保存；非法字符限制；重复词头提示 **覆盖 / 取消** |
| **删除** | 二次确认 Modal；仅 `source=user` 或策略 B 下可删隐藏项 |
| **编辑** | 同上；保存后搜索与详情立即生效 |
| **切换 Tab** | 添加表单默认分类与当前 Tab 一致 |

## 六、与现有页面的关系（对照你提供的截图）

| 现有元素 | 变化 |
|----------|------|
| 搜索栏 | 逻辑不变，检索范围 **含用户词** |

## 八、验收标准（产品侧）

1. 在「日常」「外贸」下各新增 1 条词，**列表与搜索** 均能命中。  
2. 编辑该词中文释义后，**无需杀进程** 即列表与详情一致。  
3. 删除用户词后，列表与搜索不再出现；**内置词不受影响**（在策略 A 下）。  
4. 用户词与内置词在 UI 上 **可区分**（角标或副标题）。  
5. 无网络环境下 **全部功能可用**（确认仅本地存储）。
