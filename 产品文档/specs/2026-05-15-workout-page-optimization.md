# 训练页优化 — 产品设计文档

## 一、概述

**目标：** 将当前简单的训练录入页升级为功能完整的动作浏览器 + 训练记录系统，参考训记的核心交互范式，提供专业级的训练体验。

**核心改动：**
- 新增**动作浏览器**（左侧肌群树 + 右侧器材 Tab + 动作网格）
- 新增**训练选择模式**（新建训练时选动作，浏览模式只看详情）
- 改进**训练录入**（默认 5 组、第 1 组热身标记、难度评级）
- 新增**动作详情弹窗**（图文介绍）
- 新增**动作历史**（查看某个动作的历史记录）

**原则：** 浏览 + 选择分离，新建训练才能编辑，平时浏览只读。

## 二、页面架构

### 2.1 状态流转

```
进入页面 ──→ 动作浏览器（浏览模式）
                  │
          点击「新建训练」
                  ↓
              选择模式 ←────────┐
       点动作 → + 加入已选列表     │
       底部出现「开始训练 (N)」     │
                  │              │
                  ↓              │
              训练模式            │
       默认 5 组 / 热身 / 难度    │
       新增一组 / 动作历史          │
       切换动作 / 结束训练         │
                  │              │
                  ↓              │
              完成页             │
       显示本次训练总结            │
       点击「新训练」───────────┘
```

### 2.2 顶部操作栏

```
┌──────────────────────────────────────────┐
│  ← 返回    自由训练                       │
│  [新建训练]  [历史记录]                    │
└──────────────────────────────────────────┘
```

- **新建训练：** 进入选择模式，可添加动作到本次训练
- **历史记录：** 弹出侧栏/列表，展示过往训练记录，点击查看详情
- 浏览模式下不显示「新建训练」按钮时，可用悬浮 FAB 替代

**完整分类：**

```

**交互：**
- 一级节点点击展开/折叠子分类（小三角图标）
- 子分类点击 → 右侧刷新为该分类动作列表
- 默认展开第一个肌群（胸）的第一个子分类（上胸）
- 选中状态高亮

### 3.3 右侧：器材 Tab + 动作网格

选中某个子分类后，右侧按器材类型分 Tab：

**器材 Tab 栏（固定 6 类）：**

| Tab | 器材 |
|-----|------|
| 🧲 杠铃 | Barbell |
| 🏋️ 哑铃 | Dumbbell |
| 🪢 绳索 | Cable |
| 🦾 悍马机 | Hammer Strength |
| ⚙️ 固定器械 | Machine |

**交互规则：**
- 默认选中第一个有动作的 Tab
- 空的 Tab 显示为灰色不可点
- 切换 Tab 只更新动作网格，不刷新左侧树

**动作网格卡片：**

```

- 浏览模式：整个卡片可点击 → 弹出详情
- 选择模式：右下角显示 "+" 按钮 → 点击添加到已选列表

## 四、训练流程

### 4.1 选择模式

点击「新建训练」后：

- 左侧树 + 右侧网格不变
- 每个动作卡右下角出现 "+" 按钮
- 点击 "+" → 动作添加到「已选动作列表」，卡片显示 ✓ 标记
- 底部固定栏出现：

```
┌──────────────────────────────────────┐
│ 已选 6 个动作  [查看列表]  [开始训练] │
└──────────────────────────────────────┘
```

- 点击「已选列表」→ 弹出已选动作清单，可删除
- 点击「开始训练」→ 进入训练模式

### 4.2 训练模式

```
┌──────────────────────────────────────┐
│  ← 返回    自由训练                   │
│  已完成 12 组 · 总容量 4860 kg        │  ← 进度条
├──────────────────────────────────────┤
│                                       │
│  ┌─ 当前动作 ──────────────────────┐  │
│  │ 上斜杠铃卧推                     │  │
│  │ 1/6 个动作                       │  │
│  │                                  │  │
│  │ ┌─────────────────────────────┐  │  │
│  │ │ #   重量(kg)  次数  备注     │  │  │
│  │ │ 🔥1   20      12    热身    │  │  │
│  │ │  2   40       10           │  │  │
│  │ │  3   45       8            │  │  │
│  │ │  4   45       8            │  │  │
│  │ │  5   50       6            │  │  │
│  │ └─────────────────────────────┘  │  │
│  │                                  │  │
│  │  本组感受:  ○简单  ●正常  ○困难  │  │  ← 新增
│  │                                  │  │
│  │  [重量___] × [次数___] [备注___]  │  │  │
│  │  [✓ 完成本组]                     │  │  │
│  │                                  │  │  │
│  │  [+ 新增一组]                     │  │  │  ← 新增
│  │  [📊 动作历史]                    │  │  │  ← 新增
│  │                                  │  │  │
│  │  [← 上一个]  [→ 下一个]           │  │  │
│  └──────────────────────────────────┘  │
│                                         │
│  [✓ 结束训练]                           │
└──────────────────────────────────────────┘
```

**默认规则：**
- 每个动作自动生成 5 组空行
- 第 1 组默认标记为 🔥 热身（可取消）
- 组 2-5 标记为正式组
- 「新增一组」在末尾追加第 6、7...组

**难度评级：**
- 针对当前动作的所有组（一个大组），单选：简单 / 正常 / 困难
- 选择后存储为整组难度，不细到每一组
- 数据记录：记录 RPE 值（简单=6-7, 正常=8, 困难=9-10）

**动作历史：**
- 点击弹出该动作过往所有训练记录
- 显示：日期、每组重量×次数、历史最高重量、历史最高容量
- 帮助决定当前训练的重量选择

**动作切换：**
- 点击顶部已选动作列表中的动作直接切换
- 也可用 ← → 按钮切换（同现在）
- 切换时保留各动作的独立录入状态

### 4.3 完成页

与现有设计一致，显示训练总结：
- 总组数、总容量
- 每个动作的完成情况
- 「开始新训练」按钮

## 五、数据模型变更

### 5.1 ExerciseDef 扩展

```typescript
export type MuscleGroup =
  | '胸' | '背' | '肩' | '腿' | '手臂' | '腹' | '有氧' | '全身';

export type SubMuscleGroup =
  // 胸
  | '上胸' | '中胸' | '下胸'
  // 背
  | '宽度' | '厚度'
  // 肩
  | '前束' | '中束' | '后束'
  // 腿
  | '股四头' | '股二头' | '臀'
  // 手臂
  | '二头' | '三头'
  // 腹、有氧无子分类
  | '腹' | '有氧';

export type Equipment =
  | '杠铃' | '哑铃' | '绳索' | '悍马机' | '固定器械' | '徒手' | '有氧器械';

export interface ExerciseDef {
  id: string;
  name: string;
  muscleGroup: MuscleGroup;
  subMuscleGroup: SubMuscleGroup;  // 新增
  equipment: Equipment;            // 新增
  type: ExerciseType;
  description?: string;            // 新增：动作要领
  imageUrl?: string;               // 新增：图片路径
  tags?: string[];                 // 新增：标签
  isCustom?: boolean;
}
```

### 5.2 WorkoutSet 扩展

```typescript
export interface WorkoutSet {
  weight: number;
  reps: number;
  type?: 'warmup' | 'normal' | 'drop' | 'failure';  // 新增
  note?: string;
}
```

### 5.3 WorkoutExercise 扩展

```typescript
export interface WorkoutExercise {
  exerciseName: string;
  muscleGroup: MuscleGroup;
  subMuscleGroup?: SubMuscleGroup;  // 新增
  equipment?: Equipment;             // 新增
  sets: WorkoutSet[];
  difficulty?: 'easy' | 'normal' | 'hard';  // 新增：整组感受
}
```

### 5.4 新增：ExerciseRecord（动作历史缓存）

```typescript
export interface ExerciseRecord {
  exerciseName: string;
  sessions: Array<{
    date: string;
    sets: WorkoutSet[];
    difficulty?: 'easy' | 'normal' | 'hard';
  }>;
  maxWeight: number;    // 历史最高重量
  maxVolume: number;    // 历史最高单次容量
}
```

## 七、UI 素材需求

**动作占位图：**
- 每个动作需要一个卡通风格插图（约 120×120px）
- 第一期：使用 emoji + 文字占位（如 🏋️ + 动作名首字）
- 后续：从 iconfont 下载或自行设计卡通 SVG，替换 imageUrl 字段

**器材图标：**
- 6 个器材 Tab 需要 tab 图标（杠铃/哑铃/绳索/悍马机/固定器械/徒手）
- 用 Unicode emoji 即可（🧲🏋️🪢🦾⚙️🧍）

**肌群树图标：**
- 展开/折叠箭头用系统 triangle 或 `›` 字符

## 八、实施建议

**分期实施：**

| 阶段 | 内容 | 工作量 |
|------|------|--------|
| P1 | 数据模型升级 + 预设动作库扩展 | 小 |
| P2 | 动作浏览器 UI（左树 + 右网格 + 器材 Tab） | 大 |
| P3 | 选择模式 + 训练模式改进（默认5组/热身/难度） | 中 |
| P4 | 动作详情弹窗 + 动作历史 | 中 |

P1-P2 完成后可预览动作浏览器效果，P3-P4 完善训练体验。
