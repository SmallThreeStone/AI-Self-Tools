# 微信「工具箱」类小程序：开发思路与分阶段规划

本文档面向**自用型**工具聚合小程序：统一「工具注册表 + 导航 + 本地存储」，在此之上按需增加子工具页面。默认技术选型：**微信原生 + TypeScript**（调试链路短、官方文档最全；后续若要 H5 再评估 Taro / uni-app）。

---

## 一、技术选型对比（简要）

| 方向 | 市面常见度 | 优点 | 缺点 |
|------|------------|------|------|
| **微信原生（自用首选）** | 官方示例与文档均以原生为主 | 调试链路最短、API 跟进最快 | 多端复用差 |
| **Taro（React）** | 需要顺带 H5 / 鸿蒙的团队常见 | React 心智一致；可扩展多端 | 构建与升级需跟框架版本 |
| **uni-app（Vue）** | 独立开发者与外包项目常见 | 插件多；Vue 上手快 | 部分 API 需条件编译 |

**结论**：第一期用原生 + TS；页面路由与数据尽量**配置驱动**，便于将来迁移框架。

## 二、从参考界面抽象的基础架构

### 1. 信息架构与导航

- **底部 Tab（3 个）**：`首页` | `分类` | `我的` — 在 `app.json` 中配置 `tabBar`。
- **首页**：顶部搜索；区块 **我的收藏** / **最近使用** / **猜你喜欢**；工具以 **4 列宫格** 展示；收藏项显示 **星标**。
- **分类页**：标题「全部分类」；**左侧分类列表**（`categoryId`）；**右侧** `scroll-view` 列出当前分类下的工具 **横条卡片**（图标 + 标题 + 描述 + 可选星标）。
- **我的**：账号无关的自用场景下，主要承载 **主题 / 数据导出 / 清除缓存 / 关于** 等。

### 2. 统一「工具注册表」数据模型

用 **一份配置**（如 `config/tools.ts`）驱动首页、分类、搜索，避免多处硬编码。

建议字段：

| 字段 | 说明 |
|------|------|
| `id` | 稳定唯一字符串，用于收藏、最近使用、统计 |
| `name` | 展示名称 |
| `desc` | 简短描述 |
| `categoryId` | 所属分类 |
| `path` | 小程序页面路径，如 `/package-tools/pomodoro/index` |
| `icon` | emoji 或本地图标 key |
| `tags` | 字符串数组，用于搜索与推荐 |
| `requiredScopes` | 可选，所需权限说明（相册、剪贴板等），便于审核与用户提示 |

**本地存储（`wx.setStorage`）建议键名：**

- `toolbox_favorites`：`string[]`（工具 id）
- `toolbox_recent`：`{ id: string; ts: number }[]`（去重后保留最近 N 条，如 20）
- `toolbox_theme`：`light` | `dark`
- `toolbox_usage`：按 id 统计点击次数（驱动「猜你喜欢」）

**搜索**：对 `name + desc + tags.join` 做子串匹配；可扩展 **命令别名**（见下文创意）。

### 4. 导航与分包策略

- **统一入口**：`openTool(id)` 根据注册表解析 `path`，写入最近使用（并可更新使用统计），再 `wx.navigateTo`。
- **分包**：工具多时放入 `subpackages`，降低主包体积；工具注册表里的 `path` 指向分包页面即可。

## 三、在参考界面基础上的增强创意（自用向）

1. **命令面板式搜索**：为常用工具配置别名（如 `bmi`、`pwd`），输入即跳，减少点击路径。
2. **推荐策略**：第一期可用「同分类随机 + 排除已收藏」；有使用统计后改为「按本地点击加权」。
3. **能力声明**：注册表中 `requiredScopes` 与页面 `onShow` 提示一致，减少审核与用户困惑。
4. **暗色模式 + 主题色**：CSS 变量 + `toolbox_theme` 持久化。
5. **隐私提示**：密码复制等场景提示「妥善保管，勿分享截图」。
6. **导出配置**：将收藏夹与部分工具本地配置导出为 JSON（再通过用户自选方式备份）。
7. **本地使用统计**：仅本机聚合，用于排序「猜你喜欢」。
8. **深链（按需）**：后续可查微信平台「小程序码 / URL Scheme」能力，从外部唤起指定工具页。

## 四、分阶段规划与验收标准

### 一期 — 壳子与数据打通

**目标**：三 Tab、注册表驱动首页与分类、搜索过滤、收藏与最近使用、若干占位或真实跳转。

**验收**：

- [ ] Tab 切换正常，三类页面可打开。
- [ ] 首页：搜索能过滤工具；收藏 / 最近数据读写正确；猜你喜欢有默认策略。
- [ ] 分类：左侧切换分类，右侧列表随之变化。
- [ ] `openTool` 统一写最近使用并跳转。

### 二期 — 首批低依赖工具（分包）

**目标**：番茄钟、折扣计算、单位换算、密码生成等（少权限）。

**验收**：

- [ ] 分包配置正确，首次打开工具无路径错误。
- [ ] 各页核心交互可用；必要时使用剪贴板 API 并声明用途。

### 三期 — 媒体与 Canvas 类

**目标**：如水印、文字转图等（`canvas`、相册、保存到相册）。

**验收**：

- [ ] 权限申请与失败提示完整。
- [ ] 大图注意内存与导出尺寸说明。

### 四期 — 体验与自用强化

**目标**：主题、本地统计、导出收藏、「我的」中的清理与关于。

**验收**：

- [ ] 明暗主题切换全局生效。
- [ ] 使用统计可驱动推荐排序（至少一种规则）。
- [ ] 可导出 / 导入收藏列表（JSON）；清除缓存可恢复默认。

### 五期 — 可选进阶

- 服务端 AI 搜索（自用可极简）；或继续 **离线关键词表**。
- 多端需求明确后再评估 Taro / uni-app，并保持注册表结构稳定。

## 五、纯小白上线指南：注册小程序与发布流程

以下为**个人自用同样适用**的通用路径；具体入口名称若微信改版，以 [微信公众平台](https://mp.weixin.qq.com/) 当前界面为准。

### 1. 准备材料

- **微信号**：用于管理员扫码登录。
- **邮箱**：未注册过公众平台业务的邮箱（若注册小程序账号需要）。
- **主体类型**：个人开发者选 **个人**；若有营业执照可选企业（功能权限更多，但认证费用与资料不同）。
- **小程序名称**：提前想好；一年内可有限次数改名。
- **身份证**（个人）：注册与实名认证用。

### 2. 注册小程序账号

1. 打开 [微信公众平台](https://mp.weixin.qq.com/) → 选择 **小程序** → **注册**。
2. 按向导填写邮箱、密码，邮箱激活。
3. 选择主体类型（个人或企业等），填写管理员微信扫码绑定。
4. 完成 **信息登记** 与实名流程。

### 3. 获取 AppID

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)（小程序）。
2. 在 **开发** → **开发管理** → **开发设置** 中查看 **AppID**。
3. 在本项目 **微信开发者工具** 中创建/导入工程时填入该 AppID（测试可使用测试号，正式发布需真实 AppID）。

### 4. 安装开发者工具与导入项目

1. 下载 **微信开发者工具**（Windows / macOS）。
2. 选择「导入项目」或「新建小程序」，目录指向本仓库 `Wechat_Mini_Program`（或包含 `project.config.json` 的根目录）。
3. 将 **AppID** 填入；若仅本地预览可先用测试号或游客模式（上传代码需合法 AppID）。

### 5. 本地开发与真机预览

1. 在开发者工具中编译；使用 **预览** 生成二维码，微信扫码真机调试。
2. 涉及相册、定位等能力时，在 **app.json** 里声明对应权限说明（隐私协议与接口声明按微信最新规范配置）。

### 6. 上传代码版本

1. 开发者工具右上角 **上传**，填写版本号与备注。
2. 上传成功后，代码进入公众平台「版本管理」，尚未对用户可见。

### 7. 提交审核与发布

1. 登录公众平台 → **管理** → **版本管理**。
2. 将开发版本设为 **体验版**（可选），添加体验成员扫码验证。
3. 提交 **审核**：填写类目（工具类一般选与功能匹配的类目）、小程序简介与截图；**工具聚合类**建议简介清晰说明「本地工具集合、无违规收集」等。
4. 审核通过后，点击 **发布**；发布后用户可搜索或使用小程序码访问。

### 8. 自用场景的务实建议

- **类目与描述**：如实填写功能，避免夸大或与实际不符。
- **隐私**：若仅本地存储、不上传用户数据，按表单如实勾选 **用户隐私保护指引**。
- **体验版**：上架前长期使用建议走体验版，无需审核即可迭代。
- **备案**：若涉及特定经营活动或监管要求，关注当年监管政策；纯本地工具集合一般以平台规则为准。

### 9. 常见问题

| 现象 | 可能原因 |
|------|----------|
| 无法上传 | 未使用合法 AppID、或未绑定开发者权限 |
| 审核被拒 | 类目不符、简介与功能不一致、缺少隐私说明 |
| 真机白屏 | 路径大小写、分包路径错误、ES 语法未开启增强编译 |
